# 第二十三章 可观察性

“三件套：日志、指标、分布式追踪。”小齐把新 SDK 接入两处遗留，前端以 `traceparent` 贯穿上下文，那条看不见的线，终于穿透诸黑箱。

屏幕上首次出现一条完整链路，自用户点击至后端存储，像一条被点亮的河。某节点耗时忽竖细红一根，众人追入：一处图像压缩库在边界图上退化到 O(n^2)。

“看见，才有改变的可能。”陈砺把仪表盘上的 P95/P99 换成更易懂的话：“九成九的用户，在 1.2 秒内得到反馈。”

他们在文档写一段“可观测写作指南”：写问题前，先写用户感受；写指标时，给出基线；写修复时，写明用户会如何改变。

夜色里，他也看见另一套系统——自己的——在某些时刻抖动：惧失控，握得太紧。既看见，便松手一寸，让呼吸回身。
