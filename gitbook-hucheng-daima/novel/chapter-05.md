# 第五章 路标

周一早晨十点三十，值班群里亮起红点。一个落在新构建链上的边界条件触发了回退开关，推广页在部分设备上出现了错位。

“回滚？”运营问。

“不要怪罪人，先把配置的开关打开，让受影响的路径进入轻版。”陈砺回复，“然后写一个可复现步骤出来，我们在白板上贴三句事实。”

十分钟后，订单恢复。会里几个人都发了一口气。有人提议庆功，江牧摆手：“把适配层的规则统一一下，把‘临时解决’变成‘永久做法’。”

他们把需求同源的模块放回“模板层”，把可能复用的规则抽离到“适配层”。这次调整不惊艳，却稳稳当当。两天后，陈砺在大屏上贴出一张“路标图”：

- 发布前：发布闸门 + 值守人 + 回退预案；
- 发布中：广播员 + 自诊断页；
- 发布后：故障采样板 + 三句事实 + 回流卡。

“让每个人走在同一条有路标的路上。”他说。

下午，设计同学提了一个细节：按钮的颜色在各种主题下并不统一，容易造成误解。“我们别再说‘红/绿’，统一说‘危险/成功’。”

命名像给城市换了路牌。一个个新的名字被贴上，旧的名字被归档。大家笑称这是“命名日”。

晚上，陈砺路过公司楼下的小花园。有人坐在长椅上发呆，有人在慢跑。路灯把地上的砖缝照得很清楚，他忽然觉得，这种清楚就是秩序的意义。

“路标不是命令。”他在笔记里写，“路标是关心。”
