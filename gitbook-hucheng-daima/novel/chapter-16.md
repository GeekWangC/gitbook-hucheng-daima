
# 第十六章 文档站从零到一

新文档站首页只置三物：起步、示例、常问。无花哨之饰，惟路径分明。

“写给未来的自己。”陈砺将“贡献指南”置于显眼，约定 PR 模板须连至少一处文档变更。

第一位外部贡献者来得很快，苏州同道留言：“你们的文档像说话。”

“因我们确是对一个人说话。”陈砺笑。

夜深，搜索框亮着温柔一隅。自此知识有了入口，经验不再只凭口耳相传。

第二周，他们约定了“文档日”。每周周三下午，任何人都可以暂时离开“开发”，只做三件事：写、改、问。写，是把新方法写出来；改，是把旧文档改到和现在一致；问，是帮第一次来的人把问题问清楚。文档日的规则贴在墙上：不催促，不讽刺，不打断。有人说：“这像是给知识洗澡。”

第三周，他们上线了“侧边任务栏”。任务栏不是“待办”，而是“带人”。它把“起步—示例—常见问题—术语—案例—坑与路”串成一条窄窄的带，有人称它为“手把手的带”。辛乔写了第一个“案例”：她把一次“看似简单的按钮改动”写成了三个小节：

1. 为什么要改（动机）；
2. 改了什么（方案）；
3. 可能影响了谁（影响）。

案例发出后，评论区里第一条留言是：“我终于知道‘为什么你们老说要写动机’了。”

第四周，他们做了一次“搜索词清洗”。搜索里出现频率最高的三个词是：起步、主题、日期选择。小齐把“日期选择”的词条点开，发现其中有一段“键盘导航”的解释略显粗糙，便把它换成了“人能听懂的话”。改完以后，他又给读屏器录了一段“演示音频”，放在词条的末尾。有人在评论里写：“这是我第一次在文档里‘听见’什么。”

第五周，第一位外部贡献者写下了一个“翻译词典”的提案。他希望把“术语表”翻译成另一种语言，以便他们团队内部使用。陈砺在提案下写：“欢迎，但请保留‘原文链接与注释’。”几天后，第一版翻译上线。翻译的同学在评论里写：“谢谢你们允许我用我的词。”

第六周，文档站被一次“真实的问题”推动着改变。某位新同学在“起步页”的评论里写：“我跑不起来。”他没有提供任何截图，只写了四个字。过去，大家可能会要求“补充信息”。这一次，小齐在五分钟内写了一个“排查向导”：

1. 你看见了什么；
2. 你做了什么；
3. 你得到什么；
4. 你以为为什么。

向导上线后，第二天就有三位新人成功“跑起来”。其中一位留言：“谢谢你们把‘怎么问’也教了我。”

第七周，“贡献者墙”被填得满满当当。墙上每一个名字的旁边，都挂着一行很短的“自我介绍”：有人写“我负责把一个‘看不见的错误’写成一个‘看得见的故事’”；有人写“我第一次把‘文档’当作‘产品’”；还有人写“我想让后来的人快一点”。每一行字都像一盏小灯。

第八周，陈砺把“文档写作指南”更新为“写给一个人看的话”。他把“你”放进了每一段：你会遇见什么、你要怎么走、你可以如何退回去。指南最后有一条“不要”：不要把“你们”写成“用户”。他写：“当我们写‘用户’的时候，我们往往忘了‘人’。”

第九周，他们引入了“版本对照”。每一页文档的右上角都有一个切换按钮，可以切换到“上一版”。有人质疑：“会不会太复杂？”陈砺说：“复杂是为了诚实。诚实是为了让后来的人知道‘我们也曾经那样’。”

第十周，“文档日”第一次走出办公室。他们在苏棠的店里摆了三张桌子，一张“写作台”、一张“问答台”、一张“演示台”。三张台桌的桌脚都垫了一张纸，纸上写着“把话说清楚”。那天，来了两个不是技术的朋友。他们坐在“问答台”前，问了一些看起来“很傻”的问题。陈砺回答时很慢，像在把一根细细的线一点点穿进针孔。最后，那两位朋友说：“我第一次觉得‘你们做的东西’不是遥远的技术，而是‘我可以用的东西’。”

第十一周，文档站出现了第一篇“失败的案例”。标题叫《我们曾经那样》。内容只有三段：当时如何想的、后来为什么不再那样了、我们如何避免再犯。有人留言：“谢谢你们把‘不好看’写出来。”

第十二周，辛乔把“术语表”做成了“术语—例子—一行解释”的三段式，并在“例子”里加入了“城市词”。比如在“锚点”下面，她写：“就像夜跑时的第五公里的灯，它告诉你走到了哪里，也告诉你可以继续往前。”

第十三周，文档站迎来了第一位“社区维护者”。他不是团队的人，却在评论里一天回了二十条问题。他在自我介绍里写：“我想把‘讲清楚’变成一种习惯。”陈砺给他发了一张小小的“维护者证”，证上只有一句话：

“谢谢你把门口打扫干净。”

那天夜里，陈砺合上电脑，风从窗缝里钻进来，吹动了桌角的一张便签。便签上写着：“把‘后来的人’放在第一行第一列。”他笑了一下，觉得心里很亮。

第十四周，他们做了第一次“文档演练周”。演练周里，所有 PR 必须附一段“对非技术同学的讲解录音”，时长不超过 90 秒。林越说：“我愿意当‘录音志愿者’。”那一周里，文档站像一座小小的电台：每一篇文档后面都多了一段温柔的声音。有人留言：“我闭上眼睛也能走到下一步。”

第十五周，陈砺把“讲清楚”的方法写成了一张很短的卡片：

1. 用对方的词；
2. 把‘术’翻成‘义’；
3. 把‘义’翻成可验证的句子；
4. 给一个可回退的入口。 

卡片被贴在文档站首页最不起眼的位置。有人问：“为什么不把它放到最上面？”陈砺说：“因为讲清楚应该在背景里，像空气一样。”

第十六周，文档站试点了“读书会”。不是读技术书，而是读“如何把话说清楚”的书。每个人选一小段，尝试用自己的词复述一遍，再把复述写进文档的某一页。辛乔选的是“比喻”一章，她把“锚点”比喻成“河上的灯笼”，把“回退”比喻成“桥上的回头路”。小齐说：“像写诗。”陈砺说：“像把‘术’翻成‘义’。”

第十七周，他们做了一次“远程贡献者小会”。一位在外地的小团队把他们的词典搬进了自己的项目，并反馈了三条“看不见但致命”的坏味道：词条重复、缩写含混、注释缺席。三条意见当晚就被拆成了 9 个 issue。第二天早上，“重复词条合并”的 PR 合并了，“缩写规范”的 PR 合并了，“注释模板”的 PR 也合并了。文档站的首页多了一个很小的徽标：由社区维护。

第十八周，他们开始记录“文档帮助了谁”。不是为了数字，是为了看见人。记录的格式很简单：谁、遇到了什么、我们怎么帮的、他现在走到哪里。一个月后，“被帮助的人”名单像一张温柔的海报。有人提议把它放到首页最上面，陈砺拒绝：“它应该在门口的旁边。来的人会看见，也不会觉得被展示。”

第十九周，文档站上线了“循迹”。任何人都可以一键生成“我走过的页面”的小地图，方便复盘“我为什么会迷路”。“循迹”上线后的第一周，复盘里出现了第一句有趣的话：“我以为自己在找‘日期选择’，原来我在找‘如何不迷路’。”

第二十周，他们把“文档演练周”的方法写进了方法论。方法的最后只有一句话：

“把话说清楚，不是为了让别人看见我们，而是为了让别人看见自己要走的路。”

第二天，第一位内部“文档维护者”自荐——不是工程师，是一位对业务特别熟的同学。她把“用户旅程”那一页补上了几个真实截图，注释了“为什么这样做”。

“文档不只是写给工程师的。”她说。

陈砺把她的名字写在“贡献者墙”第一行。墙的上方，他加了一句小字：感谢愿意把知识留下的人。
