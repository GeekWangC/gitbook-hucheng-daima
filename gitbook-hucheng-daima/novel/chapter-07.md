# 第七章 暴雨中的回归

午后的雨突然而猛烈。HC 收紧，项目合并，办公室里有人欢喜，有人忧虑。团队里出现了一句口头禅：“要，不要拖一拖。”

“把人和项目都当成模块，反而更容易受伤。”江牧说。说完他从抽屉里拿出一张“回归清单”。

那不是给代码的，是给团队的：

- 每周一次 Demo Day，展示看得见的进展；
- 统一 PR 模板，用“影响—证据—动作”三段结构；
- 一对一谈话，确认每个人的“轻与重”；
- 重新分配值守与广播，避免疲劳。

第三周，空气里有了轻微的晴味。窗边的叶子更绿了些，屏幕上的指针也不像之前那样沉重。回归不是把一切都归位，而是让秩序重新找到自己的步子。

晚上，他们在群里发了一个很小的公告：

“我们下周二试运行‘对齐日’。”

没有掌声，也没有反对。第二天早上，公告下多了一排勾勾。
