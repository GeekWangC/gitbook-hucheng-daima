# 第二十四章 小齐的第一次离开

[目录](../SUMMARY.md) · [术语表](glossary.md) · [时间线](timeline.md)

“我收到一个机会。”小齐坐在窗边，语气很平静，“想去做一段真正从 0 到 1 的事。”

“去吧。”陈砺只说了这两个字。随后补了一句：“把交接写成文档，写给未来的你。”

他们花了三晚，把小齐手上的模块逐条写进《交接笔记》：背景、依赖、易错点、如何验证、下一个建议。最后一页，小齐写了一段话：如果未来的你看到这里，还在纠结该不该重构，请先写一份 ADR。

告别那天，他们在公司楼下的桂花树边握手。秋风里有一点甜。小齐说：“我不确定这是不是对的，但我知道这是我想尝试的。”

“不需要确定，只需要承担。”陈砺笑。

几个月后，一个陌生的 GitHub ID 给他们的仓库提了 PR。注释里有一句话：我学会了把“快”放在秩序里。提交历史里，那种熟悉的节制一眼就能认出来。

他笑出声：小齐回来了，换了个名字。那天傍晚，他去了愚园路。苏棠端来两杯热饮，轻声说：“你总是把离开当作某种失败，其实那是生长。”

夜里回家，他把《交接笔记》另存为《给未来的自己》。扉页写：当你再次犹豫时，回来看这份笔记，看看当初你是怎么把“快”放在秩序里的。

他关上电脑时，窗外下起了小雨。雨声像极轻的白噪音，把城市的棱角磨得柔和。离开并不是空白，它留下了路径与方法，它让系统变大了一圈。
