# 第三章 源与路

开源这件事，经常被说得很浪漫。但在他们这里，它更像是一张清晰的地图：哪些是“路”，哪些是“庄稼”，哪些是“风景”。

“把‘路’开源。”江牧说，“把可复用、与业务无关的那部分抽出来，其他留在庄稼地里，跟着季节生长。”

路，是设计系统里的语义组件、工具函数、构建脚本与脚手架；庄稼，是每一个业务在当下的需求；风景，是被记录下来的方法与故事。

他们把第一个组件库推上了 GitHub。第一天没有星标，第二天也没有。第三天来了第一个 Issue：命名冲突。小齐写了一段很短的回复：

“感谢提醒。我们会在下个小版本内完成协调，并在 Release Note 中注明迁移方法。”

一周后，第一批 PR 到来，来自三个不同的头像。有人在 CI 里加了一个更严格的 lint 规则，有人把一个“危险的默认”改成了“明确触发”，还有人用一个小小的脚本，把“版本号”自动写进了页面底部。

“风从别处吹来。”陈砺说。

他们开了一个很短的会：

- 合并之前写清楚“影响—证据—动作”；
- 所有外部贡献，必须在 48 小时内得到“回流”：被接受/被搁置/被反对，附理由；
- 把“贡献者名单”挂在 README 的末尾。

晚上，陈砺在办公室外的台阶上坐了一会儿。风不大，灯不亮不暗。他忽然明白：开源不是给别人看，是给自己看。看见“路”如何被更多人走过，变得更稳。
